# Connect - 连接器与MCP

## 概述

OpenJiuwen Connect 将您的智能体和工作流的访问范围扩展到浏览器之外，使其可以从消息应用、终端和AI助手访问。它由两个主要组件组成:

- **连接器(Channels)** — 将OpenJiuwen带入人们日常工作的平台(Telegram、Slack、Email、CLI等)
- **MCP(模型上下文协议)** — 使AI助手能够自主调用您的智能体和工作流作为工具

## 为什么需要Connect?

高成本意味着低使用率。一个需要打开浏览器并导航到特定页面才能使用的AI系统，其实用性远不如可以从您当前所在位置直接访问的系统。Connect通过以下方式解决这个问题:

1. **面向用户** — 直接从消息应用或终端运行智能体和工作流
2. **面向AI助手** — 让AI工具自主使用您的OpenJiuwen能力

## 连接器(Channels)

### 什么是连接器?

连接器将OpenJiuwen带入您日常使用的应用和环境中。您无需打开浏览器，只需在当前位置输入消息，OpenJiuwen就会在同一位置响应。

### 功能特性

用户可以从任何连接器:

- **运行智能体** — 启动并继续与任何智能体的多轮对话
- **执行工作流** — 触发工作流；如需输入参数，机器人会交互式地收集它们
- **浏览资源库** — 列出和搜索智能体和工作流
- **管理会话** — 登录、登出、检查连接状态

每个用户直接在应用内使用自己的OpenJiuwen账户进行身份认证。他们的智能体、工作流和对话历史保持个人化和私密性。

## MCP(模型上下文协议)

### 什么是MCP?

MCP是一个开放标准，允许AI助手在对话期间调用外部工具。当OpenJiuwen作为MCP工具连接时，兼容的AI助手可以自主发现并运行您的智能体和工作流。

### 工作原理

您用自然语言向AI助手描述您想要什么。助手决定何时以及如何将OpenJiuwen用作工具 — 无需您输入命令或打开浏览器。

```
您: "找到入职工作流并为Alice运行它。"

AI助手: [搜索OpenJiuwen，检查工作流输入，运行它]
        "完成。已为Alice完成入职工作流。"
```

### 功能特性

通过MCP连接的AI助手可以:

- **查找智能体** — 搜索和浏览您的智能体库
- **与智能体对话** — 运行保持上下文的多轮对话
- **查找工作流** — 搜索和检查您的工作流库
- **运行工作流** — 使用适当的输入执行工作流并检索结果
- **检查状态** — 在采取行动之前验证后端可达性

### 兼容的AI助手

任何实现MCP的AI助手都可以连接到OpenJiuwen:

- **Claude Desktop**(由Anthropic提供) — 将OpenJiuwen配置为MCP服务端
- **JiuwenClaw** — 具有持久记忆的个人AI助手，将OpenJiuwen工作流用作结构化能力

## 安装

### 后端集成

部署OpenJiuwen Studio时会自动安装Connect功能:

1. **依赖项** — 所有必需的包(python-telegram-bot、slack-bolt、discord.py、botbuilder-core、mcp等)都包含在后端的`pyproject.toml`中

2. **Docker安装** — Connect模块会自动复制并在Docker镜像中配置适当的PYTHONPATH设置

3. **本地开发** — 对于本地开发，Connect模块位于项目根目录的`connect/`目录下

### 运行Connect的两种方式

根据您的部署方式，Connect可以通过两种方式运行:

#### 1. 直接运行(本地开发)

对于本地开发或直接在您的机器上运行后端:

```bash
# 首先安装依赖
pip install -r connect/adapters/channels/requirements.txt

# 然后运行连接器
python -m connect.adapters.channels.run <channel> <args>
```

**注意:** `pip install`步骤仅在直接运行时需要，因为在Docker外部依赖不会自动安装。

#### 2. Docker运行(生产部署)

在Docker中运行OpenJiuwen时，所有依赖已通过`pyproject.toml`安装:

```bash
# 无需pip install — 依赖已在容器中
docker exec -it <container_id> python -m connect.adapters.channels.run <channel> <args>
```

**注意:** 将`<container_id>`替换为实际的OpenJiuwen后端容器ID(使用`docker ps`查找)。

## 架构

### Connect客户端

Connect客户端(`connect/client/`)提供:

- **身份认证** — 安全的登录和会话管理
- **智能体操作** — 列出、搜索和与智能体交互
- **工作流操作** — 列出、搜索和执行工作流

### 适配器

Connect使用适配器集成不同的平台:

- **连接器适配器**(`connect/adapters/channels/`) — 处理消息平台集成
- **MCP服务端适配器**(`connect/adapters/mcp_server/`) — 实现模型上下文协议

### 添加新连接器

1. 在`connect/adapters/channels/platforms/`下创建新的平台目录
2. 使用Connect客户端实现平台特定的机器人逻辑
3. 为用户登录添加身份认证流程
4. 添加包含配置说明的`SETUP.md`
5. 使用真实用户和智能体进行测试

## 核心概念

### 构建一次，随处访问

在OpenJiuwen Web界面中构建的智能体和工作流会自动通过所有连接器和MCP连接可用。无需重复配置。

### Web界面是工作室

连接器和MCP用于**使用**，而非**创作**。在浏览器中构建和编辑智能体/工作流；从任何地方运行它们。

### 个人会话

在连接器中，每个用户使用自己的账户登录。他们的智能体、工作流和历史记录是隔离和私密的。
