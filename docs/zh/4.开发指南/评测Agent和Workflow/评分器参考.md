# 评分器 API 参考

评分器对单次试运行的输出进行评估，并返回 `{passed: bool, score: 0.0-1.0}`。

## 评分器类型

| 类型 | 值 | 描述 |
|------|----|------|
| DETERMINISTIC | 0 | 基于规则的检查——不调用大模型 |
| MODEL_BASED | 1 | 使用评分标准的大模型裁判 |
| CODE_BASED | 2 | 自定义 Python 函数 |

---

## 确定性评分器（`grader_type: 0`）

### `output_check` — 比较最终输出

```yaml
- name: "output_contains_success"
  grader_type: 0
  config:
    check_type: "output_check"
    condition: "contains"
    expected_value: "success"
    path: "result.status"
    weight: 1.0
```

### `state_check` — 检查输出中的嵌套值

```yaml
- name: "score_above_threshold"
  grader_type: 0
  config:
    check_type: "state_check"
    path: "metrics.accuracy"
    condition: "ge"
    expected_value: 0.8
```

### `tool_call_check` — 验证工具是否被调用

```yaml
- name: "search_tool_was_called"
  grader_type: 0
  config:
    check_type: "tool_call_check"
    expected_tools: ["search_web", "calculator"]
```

### `pattern_check` — 对完整追踪 JSON 进行正则匹配

```yaml
- name: "trace_has_routing"
  grader_type: 0
  config:
    check_type: "pattern_check"
    pattern: "IF|condition|branch"
```

### `transcript_check` — 统计工具调用次数或组件数量

```yaml
- name: "at_least_two_tools"
  grader_type: 0
  config:
    check_type: "transcript_check"
    metric: "tool_call_count"
    condition: "ge"
    expected_value: 2
```

### 条件运算符

| 条件 | 含义 |
|------|------|
| `eq` | 等于 |
| `ne` | 不等于 |
| `gt` | 大于 |
| `lt` | 小于 |
| `ge` | 大于或等于 |
| `le` | 小于或等于 |
| `contains` | 字符串包含 |
| `not_contains` | 字符串不包含 |
| `regex` | 正则表达式搜索匹配 |
| `is_not_empty` | 不为 None / 空字符串 / 空列表 / 空字典 |

## 基于模型的评分器（`grader_type: 1`）

使用大模型裁判评估输出质量。需要提供已配置模型中的 `model_id`。

```yaml
- name: "response_quality"
  grader_type: 1
  config:
    model_id: "your-model-id"
    rubric: >
      该响应正确地将主题识别为 'routing'，提供了
      清晰的路由决策，并解释了推理过程。
    passing_score: 0.7
    assertions:
      - "Response mentions the IF condition"
      - "Response routes to positive branch"
```

大模型以 JSON 格式响应：`{"passed": true/false, "score": 0.0-1.0, "feedback": "..."}`。

---

## 基于代码的评分器（`grader_type: 2`）

定义自定义 Python 评分函数，用于处理任意逻辑。

```yaml
- name: "custom_json_structure"
  grader_type: 2
  config:
    function_name: "grade"
    code: |
      import json
      def grade(trace, expected):
          output = trace.get("final_output", "")
          try:
              parsed = json.loads(output) if isinstance(output, str) else output
              has_name = "name" in parsed
              has_price = "price" in parsed
              passed = has_name and has_price
              return {"passed": passed, "score": 1.0 if passed else 0.0}
          except Exception:
              return {"passed": False, "score": 0.0}
```

### 函数签名

```python
def grade(trace: dict, expected: dict) -> dict:
    """
    参数：
        trace: {
            "final_output": any,
            "chunks": list[dict],
            "trace_id": str,
            "token_usage": dict | None,
        }
        expected: 任务的 expected_output 字典

    返回：
        {"passed": bool, "score": float}  # score 范围 [0.0, 1.0]
    """
```

该函数也可以返回纯 `bool`——系统会将其转换为 `{"passed": bool, "score": 1.0/0.0}`。

## 多评分器与评分规则

当一个任务配置了多个评分器时：

- 只有当每个 `weight` 不为零的评分器全部通过，该次试运行才被视为**通过（`passed`）**。
- 聚合**得分**按**加权平均**计算：`Σ(score_i × weight_i) / Σ(weight_i)`。
- `weight: 0` 的评分器会被执行并记录，但不参与通过/失败判定和得分聚合——适用于信息性或诊断性评分器。

每个评分器的名称、通过率、平均得分和试运行计数均可在结果视图的**评分器**标签页中查看，便于快速定位导致失败的具体判断条件。

结合确定性评分器与基于模型的评分器并指定权重的示例：

```yaml
graders_config:
  - name: "has_json_structure"
    grader_type: 0
    config:
      check_type: "output_check"
      condition: "contains"
      expected_value: "{"
    weight: 0.3
  - name: "content_quality"
    grader_type: 1
    config:
      model_id: "gpt-4"
      rubric: "The output is a valid, complete JSON with all required fields."
      passing_score: 0.7
    weight: 0.7
```

> **关于权重字段的注意事项：** `weight` 是评分器配置对象的顶层字段，而非嵌套在 `config` 内部。

如果所有评分器的 `weight` 均为 `1.0`（默认值），则聚合得分为简单平均值。
