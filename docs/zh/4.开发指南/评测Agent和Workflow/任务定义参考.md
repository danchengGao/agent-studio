# 任务模式参考

任务是评测套件中的独立测试用例。可以通过 API 内联定义，也可以使用 YAML 文件定义。

## 任务定义模式

```yaml
task_id: "unique_task_identifier"
task_name: "人类可读的名称"
description: "该任务测试的内容"

# 难度：0=EASY，1=MEDIUM，2=HARD
difficulty: 1

# 模式类型：0=ROUTING，1=CHAINING，2=PARALLELIZATION，
#            3=ORCHESTRATOR_WORKER，4=EVALUATOR_OPTIMIZER，5=MEMORY_USAGE
pattern_type: 0

tags: ["routing", "sentiment"]

trials: 3

input_data:
  message: "I love this product!"
  context: "customer feedback"

expected_output:
  branch: "positive"
  sentiment: "positive"

graders_config:
  - name: "output_contains_positive"
    grader_type: 0
    config:
      check_type: "output_check"
      condition: "contains"
      expected_value: "positive"
      weight: 1.0
```

## `input_data`

任意 JSON/字典，作为输入传递给 Workflow 或 Agent。具体的键名取决于你的 Workflow 所期望的输入格式。

## `expected_output`

作为 `output_check` 和 `state_check` 评分器的比较目标。

## `trials`

独立执行的次数。每次试运行都会获得唯一的 `conversation_id`。用于计算：
- `pass@k`：k 次试运行中至少 1 次通过的概率
- `pass^k`：k 次试运行全部通过的概率

设置为 `1` 时，pass@k = pass^1 = 简单的通过/失败判定。

## `tags`

可选的字符串标签（例如 `["math", "hard", "regression"]`）。标签随任务一起存储并显示，可用于在套件内对任务进行分类整理。

## `graders_config`

完整的评分器参考请参见 [评分器参考.md](评分器参考.md)。

## 套件级别：自定义聚合指标

自定义指标定义在**套件**级别，而非任务级别。它们是 Python 函数，在所有试运行完成后执行，接收完整的试运行结果列表作为输入。

通过 `PUT /api/v1/evaluation/update` 将其存储在套件的 `config.custom_metrics` 数组中：

```json
{
  "evaluation_id": "...",
  "space_id": "...",
  "config": {
    "custom_metrics": [
      {
        "name": "weighted_pass_rate",
        "description": "按试运行得分加权的通过率",
        "code": "def compute(results):\n    if not results:\n        return 0.0\n    return sum(r.get('score', 0) for r in results if r.get('passed')) / len(results)"
      }
    ]
  }
}
```

`compute(results)` 函数接收一个字典列表，每个字典包含以下字段：`task_id`、`passed`、`score`、`latency_ms`、`token_usage`、`error_message`、`grader_results`。

返回 `float` 表示单一值，返回 `dict` 表示多个子值。结果显示在**指标**标签页的"Custom Metrics"区域。

自定义指标也可以通过界面中评测套件标题栏的 **Σ**（sigma）按钮进行管理。
