# HTTPS 配置指南

> 本文档以华为云环境为例，指导您完成分布式部署中的HTTPS配置。

## 一、方案介绍

在分布式部署场景下，系统需要面向公众提供服务，因此必须配置HTTPS证书以确保通信安全。同时，为了保护系统免受Web攻击，我们采用华为云WAF（Web应用防火墙）进行安全防护。本方案采用以下架构：

- 通过华为云证书管理服务购买SSL证书
- 使用ELB（弹性负载均衡）替代Nginx
- 配置浏览器到ELB之间的HTTPS单向认证
- 采用「云模式-ELB接入」方式将网站接入WAF防护

### WAF接入模式说明

本方案使用「云模式-ELB接入」方式将网站接入WAF，该模式具有以下特点：

- WAF作为旁路检测系统，不直接参与流量转发，避免因额外引入一层转发而带来兼容性和稳定性问题
- ELB将流量镜像给WAF，WAF检测后将结果同步给ELB
- ELB根据WAF的检测结果决定是否将客户端请求转发到源站

### 请求流程说明

采用「ELB接入WAF防护」的组网方案，客户端请求流程如下：

1. 用户在浏览器中输入域名，客户端向DNS服务器发送域名解析请求
2. DNS服务器返回域名对应的解析地址（通常为ELB的EIP）
3. 客户端通过EIP访问ELB（弹性负载均衡）
4. ELB将流量镜像转发给WAF（Web应用防火墙）进行安全检测
5. WAF完成检测后，将检测结果同步给ELB
6. 源站服务器处理请求并返回响应给ELB
7. ELB根据WAF的检测结果，将正常流量转发给客户端

### WAF约束限制

使用「云模式-ELB接入」方式时，需注意以下约束限制：

- 仅支持华为云独享型ELB（"规格"为"应用型（HTTP/HTTPS）"）
- 不支持以下防护规则：威胁情报访问控制、网站反爬虫、网页防篡改、敏感信息泄露、扫描防护、BOT管理、大模型检测
- 需购买WAF的标准版、专业版或企业版
- 云模式-ELB接入的域名、QPS、规则扩展包的配额与云模式-CNAME接入共用
- 仅支持特定Region，具体请参考华为云官方文档

### WAF接入前提条件

在将网站接入WAF前，需满足以下前提条件：

- 已购买WAF的云模式
- 已在同一账号下购买了独享型负载均衡，且"规格"为"应用型（HTTP/HTTPS）"
- 已完成ELB的基本配置（监听器、后端服务器组等）
- 业务流量已通过ELB正常转发

## 二、配置步骤

### 1. 购买SSL证书

登录华为云控制台，通过「证书管理服务」购买正式SSL证书：

- 参考文档：[华为云SSL证书购买指南](https://support.huaweicloud.com/usermanual-ccm/ccm_01_0074.html)

### 2. 配置ELB HTTPS监听

在ELB控制台中配置HTTPS监听并上传已购买的证书：

- 参考文档：[ELB HTTPS单向认证配置](https://support.huaweicloud.com/bestpractice-elb/elb_bp_0800.html)

### 3. 将网站接入WAF防护

完成ELB配置后，需要将网站接入WAF进行安全防护：

1. **登录WAF控制台**
   - 访问华为云WAF控制台并使用账号登录

2. **添加防护网站**
   - 在左侧导航栏中，单击"网站设置"
   - 在网站列表左上角，单击"添加防护网站"
   - 选择"云模式-ELB接入"，并单击"开始配置"

3. **配置域名基本信息**
   - **ELB（负载均衡器）**：选择已创建的独享型负载均衡器
   - **ELB监听器**：选择要防护的监听器（支持选择"所有监听器"或"指定监听器"）
   - **网站名称**：自定义网站名称（可选）
   - **防护域名**：输入要防护的域名或IP（公网IP/私网IP）

4. **完成配置**
   - 确认配置信息无误后，单击"确认添加"
   - WAF将自动关联所选ELB及其监听器

5. **验证接入**
   - 配置完成后，通过浏览器访问域名验证WAF是否正常工作
   - 可以在WAF控制台的"网站设置"页面查看防护状态

- 参考文档：[通过云模式-ELB接入将网站接入WAF](https://support.huaweicloud.com/usermanual-waf/waf_01_0287.html)

## 三、注意事项

- 确保购买的SSL证书与您使用的域名完全匹配
- ELB的安全组配置需开放443端口（HTTPS默认端口）
- 配置完成后，请通过浏览器访问域名验证HTTPS是否正常工作
